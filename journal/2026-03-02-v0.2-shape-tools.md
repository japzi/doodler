## Problem Statement

Doodler v0.1 only had a freehand pen tool. v0.2 needed four geometric shape tools (rectangle, ellipse, line, arrow) that match the hand-drawn aesthetic of pen strokes — using roughjs for a sketchy look. The shapes had to integrate cleanly with the existing architecture: selectable/movable/deletable via the pointer tool, exportable via Copy SVG, and rendered generically as `<path>` elements.

## Solution Implementation

**Core approach: pre-computed rough paths.** roughjs's `RoughGenerator` runs at shape finalization (pointer-up), not at render time. The generator converts geometry into sketchy SVG path data strings stored on the object, identical in shape to how pen strokes store their `pathData`. This means:

- SceneRenderer stays generic — every object renders as `<path d={obj.pathData}>`
- SVG export stays generic — no per-type serialization branches beyond fill vs stroke
- No roughjs dependency at render time; it's purely a creation-time cost

**Stroke vs fill distinction.** Pen strokes are filled outlines (produced by perfect-freehand). Shape paths are stroked outlines (produced by roughjs). The renderer checks `obj.type !== 'pen'` and applies either `fill={color}` or `stroke={color} + fill="none"` accordingly. The same check appears in `svgExport.ts` using the SVG attribute names (`stroke` / `stroke-width` vs `fill`).

**Single hook for all shapes.** `useShapeTool()` reads `activeTool` from the store to determine which shape type is being drawn. All four shapes share the same drag interaction pattern:
- `pointerdown`: capture start point, create preview
- `pointermove`: update preview (normalizing negative width/height for rect/ellipse)
- `pointerup`: generate rough path, compute bounding box, `addObject()`, clear preview

**Lightweight drag preview.** During drag, `ActiveShapeRenderer` renders native SVG primitives (`<rect>`, `<ellipse>`, `<line>`) with a dashed stroke — no roughjs computation per frame. The sketchy path is only generated once on pointer-up.

**Arrow construction.** An arrow is a rough line plus two short rough lines for the arrowhead. The head lines are computed from the line angle ± 30° at a fixed 16px length. All three rough paths are concatenated into a single `pathData` string.

**roughjs integration:**

```ts
import rough from 'roughjs'
const generator = rough.generator()
const drawable = generator.rectangle(x, y, w, h, { roughness: 1.5, strokeWidth: 2, bowing: 1 })
const pathData = generator.toPaths(drawable).map(p => p.d).join(' ')
```

`generator.toPaths()` returns `PathInfo[]` with `d` strings — we concatenate them into a single path string for storage.

## Development Learnings

1. **Unused import caught by strict TypeScript** — `Canvas.tsx` imported `ShapeToolType` from the types module for the `shapeTools` Set type annotation, but the Set was inferred from its initializer and the import was never used. `tsc -b` flagged `TS6196: 'ShapeToolType' is declared but never used`. Fix: removed the import, since `new Set<string>(['rectangle', ...])` doesn't need the type alias.
