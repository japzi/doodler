## Problem Statement

Doodler lost all drawings on page reload. Users had no way to save work, load previous drawings, or start fresh. The app needed localStorage auto-save/restore, JSON file save/load, and a "New Drawing" action.

## Solution Implementation

**Auto-save via Zustand subscribe.** Rather than injecting persistence into every action, a single `useStore.subscribe()` listener watches for `objects` or `viewport` reference changes and writes to localStorage. On store creation, `loadDrawing()` hydrates the initial state.

```typescript
// Subscribe listener — fires on any state change, only persists when relevant
useStore.subscribe(
  (state, prev) => {
    if (state.objects !== prev.objects || state.viewport !== prev.viewport) {
      persistDrawing(state.objects, state.viewport)
    }
  },
)
```

**JSON file export/import.** `exportProject()` creates a versioned `{ version: 1, objects, viewport }` blob and triggers download via a temporary `<a>` element with `URL.createObjectURL()`. `importProject()` reads a `File` with `FileReader`, validates the `version` field, and sets store state.

**Clear drawing.** `clearDrawing` resets `objects`, `selectedIds`, `viewport`, `activeTextInput`, and `editingTextId` to defaults. The subscribe listener then persists the empty state to localStorage. A `window.confirm()` dialog guards against accidental data loss.

**Toolbar additions.** Three new buttons after a divider: New (doc+plus icon), Save (download icon), Load (upload icon). The Load button triggers a hidden `<input type="file" accept=".json">`.

## Testing/Validation

- `npm run build` passes (tsc + vite, zero errors)
- Draw objects, reload page — objects and viewport restored
- Click Save — `.json` file downloads with versioned drawing data
- Click Load — select file, drawing restored with toast confirmation
- Click New Drawing — confirm dialog, canvas clears, localStorage cleared

## Development Learnings

**Browser native drag hijacking.** After implementing persistence, the user reported: "when I try to drag, entire canvas seems to move (transparent), and revert." This was the browser's native drag-and-drop activating on the SVG element — it creates a transparent ghost image of the content, then reverts on release since there's no drop handler. The fix was adding `onDragStart={(e) => e.preventDefault()}` to the `<svg>` element in Canvas.tsx. This was likely latent before persistence but became noticeable once objects were present on initial render.
