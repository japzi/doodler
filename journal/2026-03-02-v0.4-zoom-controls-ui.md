## Problem Statement

The app supported mouse-wheel zoom (Canvas.tsx:112–127) but had no visible zoom controls. Users on touchpad-less setups or those unfamiliar with scroll-to-zoom had no way to discover or use the zoom feature. A visible UI widget was needed for discoverability.

## Solution Implementation

Added a `<ZoomControls />` component fixed to the bottom-right corner, separate from the top toolbar. The widget contains three elements:

- **"−" button** — divides scale by 1.1, clamped to min 0.1
- **Percentage label** — displays `Math.round(scale * 100) + '%'`; click resets to 100% (scale 1, offsets 0)
- **"+" button** — multiplies scale by 1.1, clamped to max 10

Zoom targets the viewport center (`window.innerWidth/2`, `window.innerHeight/2`) using the same offset recalculation formula as the existing wheel handler:

```ts
const newOffsetX = cx - (cx - viewport.offsetX) * (newScale / viewport.scale)
const newOffsetY = cy - (cy - viewport.offsetY) * (newScale / viewport.scale)
```

The component reads `viewport.scale` reactively via `useStore` for the display, and uses `useStore.getState()` inside event handlers to avoid stale closures. Constants (`ZOOM_FACTOR = 1.1`, `MIN_SCALE = 0.1`, `MAX_SCALE = 10`) match the existing wheel zoom behavior.

Styling mirrors the toolbar: white background, `border-radius: 10px`, same `box-shadow`, button colors (`#444`), and hover state (`#f0f0f0`).
