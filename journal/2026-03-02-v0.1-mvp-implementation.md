## Problem Statement

Build the first working version of Doodler — a web-based vector drawing editor that produces hand-drawn/sketchy SVGs. The MVP needed: a canvas with pan/zoom, a freehand pen tool with sketchy rendering, basic object selection/move/delete, a stroke color picker, and one-click SVG copy to clipboard. Target bundle size under 300KB gzipped.

## Solution Implementation

**Tech stack:** React 19 + TypeScript via Vite, perfect-freehand for sketchy strokes, Zustand for state, nanoid for IDs.

**Key design decisions:**

1. **SVG DOM rendering** — each drawn object is a `<path>` element in the DOM, which maps directly to the SVG export output. No separate canvas rendering pipeline needed.

2. **perfect-freehand for the doodle aesthetic** — instead of using roughjs (reserved for v0.2 shape tools), pen strokes use `getStroke()` from perfect-freehand which converts pointer input into a smooth freehand outline polygon. The output is a filled `<path>` (using `fill`, not `stroke`), which naturally looks hand-drawn.

3. **Pre-computed path data** — on pointer-up, the stroke's SVG `pathData` string is computed once and stored on the object. Rendering just reads the string; no recomputation on re-render.

4. **Object movement via translate offset** — each object has a `position: {x, y}` that applies via `transform="translate()"`. Moving objects doesn't require recomputing path data.

5. **High-frequency input bypass** — during `pointermove`, the store is updated via `useStore.getState()/setState()` directly, bypassing React's render cycle for performance.

6. **DOM-based hit testing** — each `<path>` carries a `data-object-id` attribute. On click, `event.target.closest('[data-object-id]')` lets the browser's SVG engine handle geometry hit-testing for free.

7. **Zoom toward cursor** — scroll-wheel zoom recalculates the viewport offset so the point under the cursor stays fixed: `newOffset = cursor - (cursor - oldOffset) * (newScale / oldScale)`.

8. **Clean SVG export** — serialization reads from the data model (not DOM scraping), computes a tight viewBox with padding, and outputs clean `<svg>` with `<path>` elements. Clipboard uses `navigator.clipboard.writeText()`.

## Development Learnings

1. **Vite scaffolding needed `vite-env.d.ts`** — the initial `tsc` build failed with `Cannot find module './App.css'` and similar errors for all CSS imports. Vite provides ambient type declarations for CSS modules via `/// <reference types="vite/client" />`, but the file wasn't created during manual project setup (since `npm create vite` couldn't run non-interactively). Fix: added `src/vite-env.d.ts`.

2. **Mismatched tool callback signatures** — `penTool.onPointerUp()` and `pointerTool.onPointerUp()` take zero arguments, but `Canvas.tsx` was passing the pointer event as an argument: `penTool.onPointerUp(e)`. TypeScript caught this as `Expected 0 arguments, but got 1`. Fix: removed the argument at the call site and prefixed the unused parameter with `_`.

3. **Vite v8 requires Node ≥20.19 or ≥22.12** — `npm create vite@latest` pulled v8.3.0 which rejected Node 20.9.0. Fix: pinned to `create-vite@6` / `vite@^6.0.0` which supports the current Node version.
