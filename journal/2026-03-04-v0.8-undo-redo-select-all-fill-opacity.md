## Problem Statement

All 14+ store actions that mutated `objects` were irreversible — no way to undo accidental deletes, drags, style changes, or any other operation. Additionally, there was no select-all shortcut, and opacity was applied to entire objects (stroke + fill + text) rather than just the fill.

## Solution Implementation

### Undo/Redo — Snapshot-based approach

Added `_history: SceneObject[][]` (undo stack, max 50) and `_future: SceneObject[][]` (redo stack) to the store. Three new actions:

- `saveSnapshot()` — pushes `structuredClone(objects)` onto `_history`, clears `_future`
- `undo()` — pops last history entry, pushes current objects to `_future`, restores
- `redo()` — pops last future entry, pushes current objects to `_history`, restores

Key design decision: **discrete vs continuous mutations**. Discrete mutations (addObject, deleteObjects, duplicateObjects, updateTextObject, bringForward, sendBackward, matchSize, alignObjects, distributeObjects, updateObjectStyles, clearDrawing) auto-snapshot inline — each call pushes history before mutating. Continuous mutations called every pointer-move frame (moveObjects, resizeObjects, updateArrowGeometry, updateLineGeometry, updateArrowHeadSize) do NOT auto-snapshot. Instead, the pointer tool calls `saveSnapshot()` once at drag start, so an entire drag/resize operation = one undo step.

### Select All — Cmd/Ctrl+A

Switches to pointer tool first, then selects all object IDs. Order matters because `setActiveTool` clears `selectedIds`.

### Fill-Only Opacity

Moved the SVG `opacity` attribute from the parent `<g>` and `<path>` elements down to just the `FillShape` component (`<rect>`/`<ellipse>` with fill). Removed opacity from text, pen strokes, lines, and arrows entirely. Toolbar opacity slider now only appears when a fill-capable tool is active or fillable objects are selected.

## Development Learnings

1. **Cmd+A select all didn't work initially.** Used if-statements instead of switch cases for the modifier key handler — the switch/case structure may have had fall-through issues on macOS with Cmd held. Restructured to explicit if-checks which resolved it.

2. **Select all selected objects but they immediately disappeared from selection.** User corrected: "it switches to pointer tool alright, but doesn't select all objects." Root cause was calling `setSelectedIds()` before `setActiveTool('pointer')` — the latter clears `selectedIds` as part of its implementation (`set({ activeTool: tool, selectedIds: new Set(), ... })`). Fix: call `setActiveTool` first, then `setSelectedIds`.

3. **updateArrowHeadSize initially auto-snapshotted** like other discrete mutations per the plan, but it's actually called continuously during head-size handle drag in the pointer tool. Reverted to not auto-snapshot — the pointer tool's `saveSnapshot()` at drag start handles it correctly, matching the pattern for other continuous mutations.
