## Problem Statement

1. The selection action bar used text labels ("Same Width", "Left", "Space H", etc.) which took too much space and looked inconsistent with the icon-based toolbar.
2. When editing text (double-click), the selection overlay with resize handles remained visible, cluttering the editing experience.
3. The pointer tool used a filled arrow icon that didn't match the outlined style of other toolbar icons.
4. Only bring-forward/send-backward existed — no way to jump directly to front or back of the z-order stack.
5. Z-order controls were only accessible via keyboard shortcuts (Cmd+]/[), not from the action bar.

## Solution Implementation

### Icon-based action bar
Replaced all 11 text buttons with 16x16 SVG icons. Each button is a fixed 28px square with flexbox centering. Icons use `currentColor` for consistent theming. All buttons retain `title` attributes for hover tooltips.

### Hide overlay during text edit
Added `editingTextId` subscription to `SelectionOverlay`. The selection bounding box and resize handles skip rendering when `editingTextId` is truthy: `if (selectedIds.size > 0 && !editingTextId)`.

### Z-order controls
Added `bringToFront` and `sendToBack` store actions. Both use filter-and-concat — `bringToFront` moves selected objects to end of array, `sendToBack` moves them to start. Both include undo snapshots. The action bar now shows for 1+ selected objects (z-order always visible), with match size/alignment at 2+ and distribute at 3+.

### Pointer icon
Changed from a filled arrow path to an outlined cursor icon (arrow + diagonal line) matching the stroke-based style of other toolbar icons.

## Development Learnings

1. **Cmd+A select-all didn't work initially.** Root cause was `setActiveTool('pointer')` clearing `selectedIds` internally (`set({ activeTool: tool, selectedIds: new Set() })`). Calling `setSelectedIds` before `setActiveTool` meant the selection was immediately wiped. Fix: call `setActiveTool` first, then `setSelectedIds`. User feedback: "it switches to pointer tool alright, but doesn't select all objects."

2. **Cmd+A in the keydown handler needed to be pulled out of the switch statement.** The initial implementation used a switch/case inside the `if (hasModifier)` block, but Cmd+A wasn't firing. Restructured to explicit if-statements for `z`/`a` keys before the switch for the remaining modifier shortcuts.
