## Problem statement
Undo/redo, select all, and delete all worked via keyboard shortcuts (Cmd+Z, Cmd+Shift+Z, Cmd+A, Delete/Backspace) but had no UI buttons. On mobile devices without a hardware keyboard, these actions were completely inaccessible.

## Solution implementation
**Undo/Redo buttons (`Toolbar.tsx:138–161`):** Two buttons placed at the very start of the toolbar, before the pointer tool button, separated by a divider. Each uses a standard curved-arrow SVG icon (counter-clockwise for undo, clockwise for redo). Disabled state is derived directly from the store's `_history` and `_future` arrays:
```ts
const canUndo = useStore((s) => s._history.length > 0)
const canRedo = useStore((s) => s._future.length > 0)
```
Buttons call the existing `undo` and `redo` store actions — no new logic needed.

**Select All button (`Toolbar.tsx:405–419`):** Placed in the utility section next to the grid toggle. Uses a dashed-rectangle + checkmark SVG icon. Disabled when `objects.length === 0`. On click, switches to the pointer tool and selects all object IDs:
```ts
onClick={() => {
  setActiveTool('pointer')
  const allIds = new Set(useStore.getState().objects.map((o) => o.id))
  useStore.getState().setSelectedIds(allIds)
}}
```

**Delete button (`SelectionActionBar.tsx:183–190`):** A trash-can icon button added at the end of the floating selection action bar (which already appears when objects are selected). Calls the existing `deleteObjects` store action with the current `selectedIds`. Preceded by a divider to visually separate it from the alignment/distribution controls.
