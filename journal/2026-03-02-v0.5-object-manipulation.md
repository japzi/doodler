## Problem Statement

The editor supported single-object selection, move, and delete, but lacked essential manipulation features: duplicating objects, controlling draw order, selecting multiple objects at once, and resizing. These are table-stakes for any drawing tool and were listed as the v0.5 milestone.

## Solution Implementation

### Duplicate (Ctrl/Cmd+D)

Uses `structuredClone` to deep-copy selected objects, assigns fresh IDs via `generateId()`, offsets position by +10,+10, and replaces the selection with the new clones:

```ts
duplicateObjects: (ids) =>
  set((state) => {
    const clones: SceneObject[] = []
    for (const obj of state.objects) {
      if (ids.has(obj.id)) {
        const clone = structuredClone(obj)
        clone.id = generateId()
        clone.position = { x: clone.position.x + 10, y: clone.position.y + 10 }
        clones.push(clone)
      }
    }
    return { objects: [...state.objects, ...clones], selectedIds: new Set(clones.map((c) => c.id)) }
  }),
```

### Z-Order (Ctrl+] / Ctrl+[)

`bringForward` iterates end-to-start swapping selected objects with their next non-selected neighbor. `sendBackward` does the reverse. This preserves relative order of multi-selected objects.

### Multi-Select

Two mechanisms in the pointer tool:
1. **Shift+click** ‚Äî toggles an object in/out of `selectedIds` instead of replacing
2. **Marquee drag** ‚Äî dragging on empty canvas tracks start point, updates `marqueeRect` in store on move, and on release finds all objects whose world-space bounding box intersects the marquee via `boxesIntersect(a, b)` AABB test. Shift merges with existing selection.

### Resize Handles

`SelectionOverlay` renders 4 corner handles as small white squares with blue borders, sized at `8 / viewport.scale` to stay constant on screen. Each handle carries a `data-resize-handle="nw|ne|sw|se"` attribute.

The pointer tool checks for this attribute before the object hit test. On resize drag, it computes `scaleX`/`scaleY` from (anchor‚Üícurrent) vs (anchor‚Üíoriginal corner), where anchor is the opposite corner. Shift constrains to uniform scale.

`applyResize` in `src/utils/resize.ts` dispatches by object type:
- **Rect/Ellipse**: scales world-space corners around anchor, regenerates rough path
- **Line/Arrow**: scales endpoints, regenerates rough path
- **Pen**: scales all points, recomputes bounding box and stroke path
- **Text**: scales position around anchor, scales `fontSize` by `|scaleY|` (min 8px), remeasures text bounds
