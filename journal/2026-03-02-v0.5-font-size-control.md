# v0.5 Font Size Control for Text Tool

**Date:** 2026-03-02

## Files Modified

- `src/store/useStore.ts` (L10, L33, L45, L80) — added `fontSize: number` state (default 24) and `setFontSize` action
- `src/components/Toolbar/Toolbar.tsx` (L8, L11, L106–118) — added font size `<select>` dropdown (14–64px), conditionally rendered when text tool is active
- `src/components/Toolbar/Toolbar.css` (L68–89) — added `.toolbar__font-size-select` styles matching existing toolbar aesthetic
- `src/components/Canvas/TextInputOverlay.tsx` (L7, L25, L51–57) — replaced hardcoded `DEFAULT_FONT_SIZE = 24` with store-driven `fontSize`; new text objects now use the selected size

## Problem Statement

The text tool hardcoded `DEFAULT_FONT_SIZE = 24` in `TextInputOverlay.tsx`. Users had no way to change the font size of new text objects. The TODO tracker listed "Font size control" as an outstanding v0.3 item.

## Solution Implementation

Lifted `fontSize` into the Zustand store so it's accessible from both the toolbar and the text input overlay.

In the toolbar, a `<select>` dropdown with preset sizes (14, 18, 24, 32, 48, 64px) appears only when the text tool is active, keeping the toolbar clean for other tools:

```tsx
{activeTool === 'text' && (
  <select value={fontSize} onChange={(e) => setFontSize(Number(e.target.value))}>
    {[14, 18, 24, 32, 48, 64].map((size) => (
      <option key={size} value={size}>{size}px</option>
    ))}
  </select>
)}
```

In `TextInputOverlay`, the hardcoded constant was removed. For the textarea preview, `storeFontSize` is read reactively. For committing new text objects, `useStore.getState().fontSize` is read at commit time to avoid stale closures:

```ts
const currentFontSize = useStore.getState().fontSize
const bbox = measureTextBounds(text, currentFontSize)
```

Editing existing text objects still uses the object's own `fontSize`, preserving previously placed text.
